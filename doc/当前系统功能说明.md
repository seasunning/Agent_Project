# 当前系统功能说明

## 1. 文档目的

本文档基于当前代码仓库中已经实现的功能进行整理，便于快速了解系统现状、已落地能力、页面流程、后端接口和当前产物输出范围。

---

## 2. 系统概述

`SD Multi Agents System` 是一个基于 `FastAPI + Vue 3 + Vite` 的多阶段软件开发辅助系统，当前已经实现从**需求分析**到**设计生成**再到**代码原型生成与工程落盘**的一条基础流程。

当前系统主要围绕以下阶段展开：

- **Phase 2：需求智能分析**
- **Phase 3：设计方案自动生成**
- **Phase 4：代码原型自动生成**
- **Phase 5：生成结果落盘与压缩包下载**

系统通过接入 `DeepSeek` 模型，支持结构化输出、流式返回、前端工作台展示，以及结果的导出、编辑和保存。

## 4. 当前已实现的核心功能

## 4.1 工作台首页

系统已提供统一工作台首页，用于展示当前阶段、模型接入方式、后端健康状态以及各阶段入口。

已实现内容：

- 展示系统当前阶段说明
- 展示模型接入策略
- 调用后端 `/health` 检测服务状态
- 提供需求分析、设计生成、代码生成三个页面入口
- 展示已落地能力说明

---

## 4.2 Phase 2：需求智能分析

系统已支持对用户输入的原始需求进行结构化分析。

### 已实现能力

- 输入原始需求文本
- 支持**快速模式**与**深度模式**切换
- 调用后端进行流式需求分析
- 页面实时展示：
  - 状态消息
  - 推理流内容
  - 正文内容流
- 输出结构化需求结果

### 当前结构化结果字段

需求分析结果当前包含以下字段：

- `summary`：需求摘要
- `functional_requirements`：功能需求列表
  - `name`
  - `description`
  - `priority`
- `non_functional_requirements`：非功能需求
- `constraints`：约束条件
- `actors`：参与角色
- `ambiguities`：模糊点
- `conflicts`：冲突点
- `questions_for_user`：待确认问题
- `raw_model_output`：模型原始输出
- `reasoning_content`：模型思考内容
- `mode`：分析模式

### 页面侧已实现交互

- 示例需求一键填充
- 流式分析进度提示
- JSON 导出
- Markdown 导出
- 结果存入前端工作区状态

### 后端支持接口

- `POST /api/v1/requirements/analyze`
- `POST /api/v1/requirements/analyze/stream`

---

## 4.3 Phase 3：设计方案自动生成

系统已支持基于 Phase 2 的结构化需求结果生成架构设计方案。

### 已实现能力

- 基于需求结果发起设计生成
- 支持**快速模式**与**深度模式**
- 设计过程支持流式输出
- 页面实时展示状态、推理流与内容流
- 输出结构化设计方案

### 当前设计结果字段

设计结果当前包含以下字段：

- `architecture_style`：架构风格
- `architecture_rationale`：架构说明/设计理由
- `modules`：模块划分
  - `name`
  - `responsibility`
  - `inputs`
  - `outputs`
- `apis`：接口定义
  - `name`
  - `method`
  - `path`
  - `description`
- `data_entities`：数据实体
  - `name`
  - `fields`
- `mermaid`：Mermaid 图描述
- `raw_model_output`：模型原始输出
- `reasoning_content`：模型思考内容
- `mode`：生成模式

### 页面侧已实现交互

- 设计结果展示
- Mermaid **图形视图 / 代码视图**切换
- Mermaid 代码复制
- 设计结果 JSON 导出
- 设计结果 Markdown 导出
- 设计结果**可视化编辑**：
  - 编辑架构风格
  - 编辑架构说明
  - 编辑模块
  - 编辑接口
  - 编辑实体
  - 编辑 Mermaid 图描述
- 支持恢复原始设计结果
- 支持本地草稿自动保存与恢复

### 后端支持接口

- `POST /api/v1/design/generate`
- `POST /api/v1/design/generate/stream`

---

## 4.4 Phase 4：代码原型自动生成

系统已支持基于设计方案进一步生成代码原型结果。

### 已实现能力

- 基于设计结果发起代码生成
- 支持**快速模式**与**深度模式**
- 支持模型先进行**智能技术选型**
- 支持流式返回代码生成过程
- 输出项目摘要、技术栈、文件树、关键源码文件、启动步骤

### 当前代码生成结果字段

- `project_summary`：项目摘要
- `tech_stack`：技术栈列表
- `file_tree`：项目文件树文本
- `files`：关键源码文件列表
  - `path`
  - `language`
  - `description`
  - `content`
- `startup_steps`：启动步骤
- `raw_model_output`：模型原始输出
- `reasoning_content`：模型思考内容
- `mode`：生成模式

### 当前代码生成流程特点

当前后端代码生成采用两阶段思路：

1. **先规划**项目摘要、技术栈、关键文件路径、启动步骤
2. **再逐个生成**关键源码文件内容

### 技术选型能力

当前前端已支持选择或生成以下技术选项：

- 编程语言
- 后端框架
- 前端框架
- 数据库

支持方式：

- 用户手动选择
- 用户输入自定义选项
- 调用模型进行智能推荐

### 页面侧已实现交互

- 智能选型流式输出展示
- 技术选型结果回填
- 代码生成进度展示
- 项目摘要展示
- 技术栈展示
- 启动步骤展示
- 文件树展示与复制
- 类 IDE 的关键文件浏览器
- 单文件源码预览
- 单文件代码复制
- 代码生成结果导出为 JSON
- 代码生成结果导出为 Markdown

### 测试预览能力

当前系统还提供代码生成**测试预览**能力，用于在不执行完整生成的情况下输出占位文件与预览结果。

### 后端支持接口

- `POST /api/v1/codegen/suggest`
- `POST /api/v1/codegen/suggest/stream`
- `POST /api/v1/codegen/generate`
- `POST /api/v1/codegen/generate/stream`
- `POST /api/v1/codegen/preview`

## 4.5 Phase 5：工程落盘与压缩包下载

虽然首页文案仍以 Phase 4 为主，但当前代码中已经实现生成结果写入本地目录和压缩包下载能力，可视为 Phase 5 的基础能力已落地。

### 已实现能力

- 将代码生成结果写入 `generated_projects` 目录
- 自动创建项目目录
- 自动写入关键源码文件
- 自动补充基础工程文件
- 自动生成启动说明文件
- 自动打包为 zip 压缩包
- 支持返回已存在工程的落盘信息
- 支持下载生成的压缩包

### 当前落盘附加产物

除模型生成文件外，系统还会自动补充：

- `README.md`
- `.env.example`
- `requirements.txt`（当语言/框架匹配 Python 场景时）
- `package.json`（当存在前端框架时）
- `STARTUP_STEPS.txt`
- `.zip` 压缩包

### 命名与保存策略

- 支持自定义项目名称
- 若不填项目名，会从项目摘要中自动提取名称
- 若目录重名，会自动追加 `(1)`、`(2)` 等后缀
- 当前保存策略固定为“另存为新目录”

### 后端支持接口

- `POST /api/v1/codegen/persist`
- `GET /api/v1/codegen/archive/{archive_name}`

## 5. 当前页面与流程说明

### 5.1 页面路由

当前前端已实现如下路由：

- `/`：工作台首页
- `/requirements`：需求分析页
- `/design`：设计方案页
- `/codegen`：代码生成页

### 5.2 典型使用流程

1. 在需求分析页输入原始需求并生成结构化需求结果
2. 在设计方案页基于需求结果生成系统设计方案
3. 在代码生成页选择技术栈并生成代码原型
4. 将生成结果落盘到本地目录
5. 下载打包后的 zip 工程

## 7. 当前系统边界说明

根据现有代码，当前系统已经完成的是“软件开发辅助生成”的基础链路，但仍以**原型生成**和**工作流串联**为主。

### 当前已经具备

- 结构化需求分析
- 架构设计生成
- Mermaid 图描述生成与渲染
- 技术选型推荐
- 关键代码文件生成
- 本地工程目录写入
- zip 压缩下载
- 多阶段页面串联
- 流式输出体验

### 当前更适合描述为

- 多阶段开发辅助系统
- 需求/设计/代码原型生成平台
- 面向软件项目原型产出的智能工作台

### 当前尚未看到完整落地的方向

以下内容在文案或注释中有提及，但从当前主流程代码看，尚未形成完整独立模块：

- 真正的多智能体编排引擎
- LangGraph 级别工作流调度
- 更细粒度的项目文件全量生成
- 一键写回任意外部工作区
- 更完整的权限、登录、用户体系
- 任务历史、项目管理、版本管理

因此，当前系统更准确的定位是：

> **一个已经实现需求分析、设计生成、代码原型生成与工程落盘能力的多阶段软件开发辅助系统原型。**
